# Спецификация к информационной системе "ReadMe"

## Проектная версия

# Назначение и цели создания информационной системы

## Назначение информационной системы

Система "ReadMe" предназначена для:

* преподавателей, которые хотят удобно выкладывать материалы своих курсов и привязывать к материалам предоставленную систему проверки.   
* обучающихся, которые хотят получить новые знания, с дополнительной мотивацией соревнования за счёт достижений и полностью проверить полученные знания. 

## Цели создания информационной системы

"ReadMe" создаётся в связи с запросом преподавателей факультета ИВТ ЯрГУ (например, А.Ю. Полетаев) на появление продукта, лишенного ряда недостатков текущих информационных систем (например, неудобный дизайн и архитектура, недостаточная гибкость системы проверки знаний, недостаточная вовлеченность в процесс изучения материалов). 

# Формат информационной системы

Система представляет собой веб\-приложение, доступное через веб\-браузер. Клиентскими устройствами могут выступать как настольные персональные компьютеры, так и мобильные устройства.

# Функциональные требования к информационной системе “ReadMe”

## Роли в информационной системе “ReadMe”

Каждый пользователь информационной системы имеет одну из четырёх ролей: 

* незарегистрированный пользователь;  
* обучающийся;   
* преподаватель;  
* администратор системы.

## Незарегистрированный пользователь

* “Как незарегистрированный пользователь, я хочу зарегистрироваться, чтобы получить полный доступ к функциональности ресурса.”  
* “Как незарегистрированный пользователь, я хочу получить доступ к курсам с открытым доступом, чтобы просматривать информацию, добавленную в них.”

## Обучающийся

* “Как обучающийся, я хочу просматривать и отмечать наиболее важные доступные для меня курсы, чтобы взаимодействовать с информацией, добавленной в них.”  
* “Как обучающийся, я хочу создавать, изменять и сохранять отдельный конспект к каждой теоретической теме, чтобы улучшить понимание и запоминание материала.”  
* “Как обучающийся, я хочу получать достижения за освоение курса, чтобы отслеживать собственный прогресс и соревноваться с прогрессом других.”

## Преподаватель

* “Как преподаватель, я хочу создать собственный курс, чтобы другие пользователи могли взаимодействовать с ним.”  
* “Как преподаватель, я хочу настроить видимость своего курса (публичный / приватный), чтобы регулировать доступ пользователей к нему.”  
* “Как преподаватель, я хочу предоставить к своему приватному курсу другим пользователям доступ по ссылке, чтобы регулировать доступ пользователей к нему.”  
* “Как преподаватель, я хочу добавлять темы к своему курсу, чтобы другие пользователи могли ознакомиться с ними и проверить свои знания.”  
* “Как преподаватель, я хочу при создании темы в своем курсе иметь возможность использовать автоматически-компилируемые вставки на UML, Markdown, TeX, чтобы улучшить восприятие материала пользователем.”  
* “Как преподаватель, я хочу создавать критерии для автоматического присвоения достижений пользователям и иметь возможность вручную присваивать достижения пользователям, проходящим мой курс, чтобы они могли отслеживать собственный прогресс.”  
* “Как преподаватель, я хочу удалить свой курс, поскольку он утратил свою актуальность.”  
* Все требования студента.

## Администратор информационной системы

* “Как администратор, я хочу управлять правами других пользователей (право на создание курсов, право на доступ к курсу), чтобы управлять их возможностями по взаимодействию с информационным ресурсом.”  
* “Как администратор, я хочу удалить любой курс в системе или любую тему в курсе, чтобы осуществлять их модерацию.”  
* Все требования преподавателя.

Основные страницы системы  
Для всех страниц подразумевается “Footer” с общей информацией о веб\-приложении.

1. Для неавторизованного пользователя:  
* Основная страница \- список открытых курсов без записи. Постраничный вывод списка курсов и возможность их открыть.  
* Страница курса \- материалы курса. Только демонстрация материалов курса.   
* Страница авторизации \- форма для входа в аккаунт, с возможностью вернуться на основную страницу или перейти к форме регистрации.  
* Страница регистрации \- форма для регистрации и входа в аккаунт, с возможностью вернуться на основную страницу или перейти к форме авторизации.  
* В “Header” кнопка авторизации/регистрации и кнопка перехода на основную страницу.  
2. Для авторизированного пользователя:  
1) Обучающийся:  
* Основная страница \- страница с курсами пользователя. Постраничный вывод списка курсов, на которые записан пользователь и возможность их открыть и добавить в избранное.  
* Страница поиска курсов \- для записи на новые для пользователя курсы. Постраничный вывод списка доступных курсов, на которые не записан пользователь и возможность их открыть.  
* Страница с конспектами \- список отображения конспектов пользователя. Постраничный вывод списка написанных пользователем конспектов.  
* Страница с профилем \- основная информация о пользователе. С возможностью смены пароля и просмотра достижений.  
* Страница с рейтингом пользователей по достижениям \- постраничное отображение списка пользователей, отсортированного по количеству полученных достижений.   
* Страница (окно) уведомлений \- демонстрация объявлений курсов.   
* Страница курса \- отображение курса с возможностью написания конспекта и переходом на систему проверки.   
* Страница конспекта \- отображение конспекта. С возможностью редактирования и удаления.   
* “Header” используется для навигации по веб\-приложению, а также имеет кнопку для выхода.  
2) Преподаватель:   
* Страницы студента.   
* Страница создания курса \- форма для создания курса. С возможностью добавления достижений и доступной системы проверки.  
* Страница курса с расширенным функционалом \- редактирование курса, возможности создавать события/уведомления, удаления курса.  
* Страница созданных курсов \- постраничное отображение курсов, созданных преподавателем и списком записанных на них людей, с возможностью добавления пользователей на курс или удаления людей с курса, а также выдачи достижений для них.  
3) Администратор системы:  
* Страницы преподавателя.   
* Страница всех пользователей \- постраничный список всех пользователей. С возможностью редактировать роль пользователя.

# Нефункциональные требования к информационной системе “ReadMe”

## Сопровождаемость

* Покрытие кода тестами \>= 60%;  
* Документирование API (OpenAPI 3.x);  
* Фиксирование архитектурных решений по проекту;

## Надежность

* SLA two nines (99%);   
* RTO \<= 24 ч.;   
* RPO \<= 12 ч.

## Производительность

* Открытие страниц с уроками \<= 200 мс;  
* Сохранение нового урока \<= 1000 мс;  
* До 500 одновременных пользователей;

## Безопасность

* Защита от DDoS  
* Соответствие ФЗ-152

## Среда выполнения

* PostgreSQL 18;  
* Python 3.12  
* FastAPI
